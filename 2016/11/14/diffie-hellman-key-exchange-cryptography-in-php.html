Diffie-Hellman Key Exchange Cryptography in PHP

Diffie-Hellman adalah sebuah metode kriptografi yang digunakan untuk mengirimkan secret key secara aman dengan perhitungan matematis. D-F merupakan kriptografi modern yang banyak digunakan. Lantas apa yang membuatnya mumpuni?

Published Nov 14, 2016
Scroll

Beberapa hari lalu, saya di kantor mendapat sebuah project dimana implementasi logic dan prosedural dikirimkan via API. Dalam satu sesi, yaitu sesi authentication. Kita setuju untuk menggunakan authentication via header (Bearer key). Di tengah perbincangan, salah seorang manager bertanya: “Bearer key ini aman gak sih kita lempar via network?” Well sebenernya aman aja sih selama dikirim via SSL. Atau kalau mau lebih advance, kirim menggunakan JWT. Tapi pertanyaan ini sedikit menggelitik saya.

Mungkinkah perbincangan privat dapat dipahami oleh kedua belah pihak, jika TIDAK ada dari antara mereka yang bertukar key?

Setelah baca-baca resource di sana-sini, akhirnya saya menemukan jawaban. Ya, Diffie-Hellman adalah solusinya. Ingin tau bagaimana algoritma ini bekerja? Check this out!

Gambaran Umum

Secara umum, Diffie-Hellman algoritma digambarkan seperti berikut:

TL;DR

<?php

class DiffieHellman
{
    /**
     * Our base-prime
     *
     * @var integer
     */
    protected $base = 5;

    /**
     * Our prime
     *
     * @var integer
     */
    protected $prime  = 23;

    /**
     * Class constructor
     */
    public function __construct()
    {
        if (!extension_loaded('bcmath')) {
            throw new RuntimeException('BC Math extension not loaded!');
        }
    }

    /**
     * Run this to know what happens.
     *
     * @return void
     */
    public function hello()
    {
        $alfaSecretKey = 6;  // Secret key, tidak akan dishare!
        $megaSecretKey = 15; // Secret key, tidak akan dishare!

        // Modular exponentation
        $alfaPublicKey = $this->calculatePublicKey($alfaSecretKey); // Alfa mengirimkan kode ini kepada mega
        var_dump('Alfa mengirimkan kode ini kepada Mega: '.$alfaPublicKey);

        $megaPublicKey = $this->calculatePublicKey($megaSecretKey); // Mega mengirimkan kode ini kepada Alfa
        var_dump('Mega mengirimkan kode ini kepada alfa: '.$megaPublicKey);

        $alfaSaltKey = $this->calculateSaltKey($megaPublicKey, $alfaSecretKey); // Alfa menghitung kode yang dikirimkan oleh Mega
        $megaSaltKey = $this->calculateSaltKey($alfaPublicKey, $megaSecretKey); // Mega menghitung kode yang diberikan Alfa

        // Pesan Alfa kepada Mega, tidak dikirimkan, akan di-decrypt terlebih dahulu
        $plainMessage = 'Hari yang indah, langitnya biru cerah.';

        // Alfa melakukan encrypting pada pesan yang akan dikirimkan
        $encryptedMessage = $this->encrypt($alfaSaltKey, $plainMessage);

        // Mega mendapatkan pesan dari Alfa, dan melakukan decrypt pada isi pesan
        $decryptedMessage = $this->decrypt($megaSaltKey, $encryptedMessage);

        var_dump('Pesan yang dienkripsi oleh Alfa yang diterima oleh Mega '.$encryptedMessage);

        // Berikut adalah pesan yang didapatkan oleh Alfa
        var_dump('Mega men-decrypt pesan dari Alfa: "'.$decryptedMessage.'".');
    }

    /**
     * Encrypt a message.
     *
     * @param  int $sharedKey
     * @param  string $message
     *
     * @return byte
     */
    protected function encrypt($sharedKey, $message)
    {
        $md5ofSharedKey = md5($sharedKey);

        return mcrypt_encrypt(
            MCRYPT_RIJNDAEL_256,
            $md5ofSharedKey,
            $message,
            MCRYPT_MODE_CBC,
            md5($md5ofSharedKey)
        );
    }

    /**
     * Decrypt a message.
     *
     * @param  int $sharedKey
     * @param  byte $message
     *
     * @return string
     */
    protected function decrypt($sharedKey, $message)
    {
        $md5ofSharedKey = md5($sharedKey);

        return rtrim(
            mcrypt_decrypt(
                MCRYPT_RIJNDAEL_256,
                $md5ofSharedKey,
                $message,
                MCRYPT_MODE_CBC,
                md5($md5ofSharedKey)
            ),
            "\0"
        );
    }

    /**
     * Convert private key to a public key.
     *
     * @param  int $mine
     *
     * @return int
     */
    protected function calculatePublicKey($mine)
    {
        return bcpowmod($this->base, $mine, $this->prime);
    }

    /**
     * Determine salt key based on their public key and my public key.
     *
     * @param  int $their
     * @param  int $mine
     *
     * @return int
     */
    protected function calculateSaltKey($their, $mine)
    {
        return bcpowmod($their, $mine, $this->prime);
    }
}

(new DiffieHellman)->hello();

Implementasi Melalui Matematika

Cara termudah dan yang paling autentik dalam implementasi algoritma ini adalah dengan menggunakan multicaptivate group of integers modulo p. Dimana p adalah bilangan prima, dan g adalah akar primitif modulus p. Sebagai contoh:

  1. Alfa dan Mega sama-sama setuju untuk menggunakan bilangan prima 23 dan basis modulo 5. Dalam hal ini berarti, p = 23 dan g = 5. Dimana 5 termasuk dalam akar primitif modulo 23.
  2. Alfa memilih angka 6 sebagai secret key-nya. Berarti a = 6. Kemudian Alfa mengirimkan sebuah public key kepada Mega dengan kalkulasi: A = g6 mod p. Dengan begini, nilai yang diperoleh adalah A = 56 mod 23 = 8. Singkatnya, angka 8 ini akan dikirimkan oleh Alfa kepada Mega nantinya.
  3. Di lain sisi, Mega memilih angka 15 sebagai secret key-nya. Berarti b = 15. Kemudian Mega juga mengirimkan sebuah public key kepada Alfa dengan kalkulasi yang sama, yaitu: B = g15 mod p. Dengan begini, nilai yang diperoleh adalah B = 515 mod 23 = 19, dimana angka 19 ini akan dikirimkan via network kepada Alfa nantinya.
  4. Di suatu sistem, Alfa ingin mengirimkan pesan kepada Mega. Di sini, Alfa membutuhkan public key milik Mega. Mega mengirimkan public key-nya, yaitu angka 19, kemudian Alfa melakukan perhitungan sebagai berikut s = Ba mod p. s = 196 mod 23 = 2. Dari sini, Alfa akan menggunakan salt = 2, untuk melakukan encryption pesannya.
  5. Mega yang mendapatkan pesan ini, tidak bisa meminta plain salt kepada Alfa. Dia harus meminta kepada public key milik Alfa. Dimana sebelumnya telah didapatkan public key milik Alfa adalah 8. Untuk mengetahui salt yang digunakan oleh Alfa, Mega melakukan perhitungan serupa: s = Ab mod p. s = 815 mod 23 = 2. Mega kemudian mendapatkan salt = 2 melalui kalkulasi ini. Dengan salt serupa, maka Mega dapat melalukan decrypting.

Dari kasus ini dapat disimpulkan peraturan dari kriptografi ini sebagai berikut:

  1. Prime number: 23. TIDAK dikirim via jaringan. SEMUA pemeran mengetahui angka ini.
  2. Basis modulo: 5. TIDAK dikirim via jaringan. SEMUA pemeran mengetahui angka ini.
  3. Alfa secret key: 6. TIDAK dikirim via jaringan. HANYA Alfa yang mengetahui angka ini.
  4. Alfa public key: 8. DIKIRIM via jaringan. SEMUA pemeran mengetahui angka ini.
  5. Mega secret key: 15. TIDAK dikirim via jaringan. HANYA Mega yang mengetahui angka ini.
  6. Mega public key: 19. DIKIRIM via jaringan. SEMUA pemeran mengetahui angka ini.
  7. Salt key: 2. TIDAK dikirim via jaringan. SEMUA pemeran BISA mengetahui angka ini HANYA JIKA masing-masing pemeran menggunakan Prime dan Basis yang sama.

Implementasi Melalui Pemrograman

Saya akan mencoba membuat implementasi sederhana mengenai kriptografi ini dengan PHP. Pastikan module bcmath sudah loaded.

  • Mula-mula tentukan bilangan prima dan nilai basis modulo yang digunakan.
$prime = 23;
$base = 5;
  • Kemudian tentukan masing-masing secret key yang akan digunakan.
$alfaSecretKey = 6;
$megaSecretKey = 8;
  • Hitung masing-masing public key. Public key ini akan dikirimkan melalui API.
$alfaPublicKey = bcpowmod($base, $alfaSecretKey, $mod);
$megaPublicKey = bcpowmod($base, $megaSecretKey, $mod);
  • Setelah masing-masing pemeran mendapatkan public key-nya via network, hitung salt key untuk kebutuhan encrypt / decrypt:
$alfaSaltKey = bcpowmod($alfaPublicKey, $megaPublicKey, $mod);
$megaSaltKey = bcpowmod($megaPublicKey, $alfaPublicKey, $mod);
  • Decrypt pesan Alfa menggunakan salt key yang ada
$message = 'Hari yang indah, langitnya biru cerah.';

$encryptedMessage = mcrypt_encrypt(
  MCRYPT_RIJNDAEL_256,
  md5($alfaSaltKey),
  $message,
  MCRYPT_MODE_CBC,
  md5(md5($alfaSaltKey))
);
  • Mega memperoleh pesan dari Alfa kemudian dia decrypt mengunakan salt key yang ada:
$decryptedMessage = rtrim(
  mcrypt_decrypt(
    MCRYPT_RIJNDAEL_256,
    md5($megaSaltKey),
    $encryptedMessage,
    MCRYPT_MODE_CBC,
    md5(md5($megaSaltKey))
  ),
"\0");
  • Hasil $decryptedMessage pasti akan sama dengan $message.

Consideration

Mungkin kalian berpikir, bagaimana jika memang ada seorang ‘Eve’ (Eavesdropper) yang melakukan sniffing, dan mendapatkan salah satu public key dari pemeran? Mudahnya, Eve harus mengetahui basis modulo dan bilangan prima yang digunakan oleh pemeran. Jika memang dapat diketaui, Eve harus melakukan perhitungan untuk menebak secret key yang digunakan. Bayangkan saja jika Eve mengetahui:

  1. Prime number: 23.
  2. Basis modulo: 5.
  3. X public key: 8.

Tugas dia selanjutnya adalah mengetahui X secret key. Hal ini dapat dilakukan dengan membuat script sederhana sebagai berikut:

$limit = 20; // perbesar limit jika diperlukan
$eavesDroped = 8; // public key yang dikirim over network

for ($i = 0; $i < $limit; $i++) {
    if ((int) bcpowmod(5, $i, 23) === $eavesDroped) {
        var_dump('Salah satu dari pemeran menggunakan Private Key: ' . $i);
        exit();
    }
}

Jika angka yang didapatkan memang mudah ditemukan dengan sedikit kesabaran. Namun apa jadinya jika public key yang dia dapatkan:

$eavesDroped = 128378792730192804398340;

Ya Eve akan kesulitan untuk memecahkan sandinya, karena dia butuh waktu yang sangat lama. Kurang lebih sekitar 40.708.648.126.012 tahun. Inilah yang dinamakan discrete logarithm problem. Percayalah, untuk menebak angka 23 dan 5 saja sudah sulit sekali dilakukan. Apalagi jika public key-nya sebesar ini. Jika masih kurang aman, kamu sendiri bisa menambahkan fitur lain, semisal:

  1. Prime number: .
  2. Basis modulo: .
